Strict access status

Strict workspace-only access is not enabled yet. The current release only provides dry-run reporting. Do not apply strict production migration until every active board has at least one owner, all active members can sign in, and dry-run warnings are clean.

SQL order

  1. supabase/full-suite-schema.sql
  2. supabase/trello-storage.sql
  3. supabase/trello-workspace-schema.sql
  4. supabase/trello-auth-schema.sql
  5. Strict access migration only after dry-run is clean.

Rollback rule

If a strict access migration is applied later and users lose access unexpectedly, run the rollback SQL draft under supabase/trello-strict-access-rollback.sql, then reopen the Trello page with a new cache-busting query string.
