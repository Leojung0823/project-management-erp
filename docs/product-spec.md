# 專案管理 ERP 產品規格書

## 1. 產品目標

建立一套給設計、行銷、工程、材料貿易與內部營運使用的專案管理 ERP。老闆最關心的資訊要能一眼看到：時間、預算、待辦、風險、應收帳款。

## 2. 第一階段 MVP

### 儀表板

- 進行中專案數
- 逾期任務數
- 總帳款
- 未收款
- 預估毛利
- 風險與急件
- 本週任務

### 專案中心

- 專案代號
- 專案名稱
- 客戶
- 負責人
- 預算
- 成本
- 進度
- 狀態
- 優先級
- 截止日
- 備註

### 任務控管

- 任務名稱
- 所屬專案
- 負責人
- 狀態
- 優先級
- 到期日
- 預估工時

### 客戶資料庫

- 公司名稱
- 聯絡窗口
- 電話
- Email
- 產業
- 關聯專案數

### 財務帳款

- 帳款名稱
- 所屬專案
- 金額
- 狀態
- 到期日

## 3. 第二階段正式化

- 使用者登入
- 權限控管
- 資料庫
- 檔案上傳
- 報價單版本控管
- 合約管理
- 請款與收款提醒
- 專案留言
- LINE / Email 通知
- 匯出 Excel / PDF

## 4. 權限建議

| 角色 | 權限 |
|---|---|
| superadmin | 全部資料、系統設定、刪除權限 |
| admin | 管理專案、客戶、任務、財務 |
| project_manager | 管理自己負責的專案與任務 |
| finance | 查看專案、管理請款與收款 |
| staff | 查看被指派任務、回報進度 |
| client_viewer | 僅查看指定專案進度，不看成本與內部備註 |

## 5. 開發順序

1. 建立資料庫與登入
2. 專案 CRUD
3. 任務 CRUD 與看板
4. 客戶 CRM
5. 帳款模組
6. 報表與匯出
7. 權限與通知
8. 部署與備份
